    node框架之Koa(一)

    谁养掉毛的猫 2023/4/27 nodemysqlkoa

    Koa是Express原班人马打造的新生代Node.js Web框架，Koa本身只是一个简单的中间件框架，想要实现一个足够复杂的Web应用还需要很多周边 生态支持，本文不会涉及源码等原理上的分析，旨在如何去使用Koa及周边生态去开发Web应用 Koa2中文文档 (opens new window)

    # 预备知识

    • JavaScript语言基础知识（包括ES6+语法）
    • Node.js基础知识，npm的基本使用
    • TypeScript基础知识，只需了解简单的类型注解
    • mySql基础知识，对数据库有一定的了解，会使用Navicat工具

    # 所用技术

    • Node.js：10.x及以上
    • npm：6.x及以上
    • Koa：2.x
    • MySql：推荐稳定的5.7版本及以上

    # 初始化代码

    # 项目初始化与koa的安装

    新建文件夹---koa_test(可任意起)，VScode打开该文件夹，打开控制台

    1. npm初始化
    npm init

    可一路回车，最后生成package.json文件

    1. 安装Koa框架
    npm install koa

    在根目录下创建src文件夹，并在src下创建main.js,作为项目主入口

    在根目录下创建.gitignore(git代码提交可忽略提交某些文件，如需要，请先执行git init),创建README.md文件（项目的markdown说明文件）

    1. 编写一个简单的程序
    const Koa = require('koa')

const App = new Koa();

App.use((ctx, next) => {
  ctx.body = "hello world"
})

App.listen(3000, () => {
  console.log("serve is running on http://localhost:3000");
})

    App.listen方法是对http模块的封装，当调用App.listen方法就会返回一个http.Server对象,http.createServer方法用来创建一个HTTP服务器

    const server = http.createServer(app)
server.listen(8080)

    App.use方法内只能跟函数，ctx参数代表上下文，返回数据写到body中，next为执行下一步，即next()，我们还可以通过ctx.url相当于ctx.request.url拿到请求 路由，ctx.status响应状态码。

    具体api可查看 Context Api文档 (opens new window)

    1. 执行

    我们在控制台执行node src/main.js,会看到serve is running on http://localhost:3000,浏览器中打开

    # 项目基本优化

    如果我们每次改动文件都需要执行一遍命令，这是不友好的，所以我们需要一个自动重启服务来帮助我们，引入插件nodemon

    1. 自动重启服务 安装nodemon,因为是开发依赖，所以我们需要安装到开发环境
    npm i nodemon -D
    1. 在package.json中编写命令
    "scripts": {
  "serve": "nodemon ./src/main.js",
  "test": "echo \"Error: no test specified\" && exit 1"
},

    随后执行npm run serve

    1. 读取配置文件

    在项目中，环境变量，端口号等并不是写死的，而是通过一个配置文件去统一管理与修改,插件dotenv可以很好的解决

    npm install dotenv

    在根目录中创建.env文件 写入环境变量

    APP_PORT = 8001

    为了方便统一管理，可以在src下创建config文件夹，创建config.default.js,引入dotenv并导出

    const dotenv = require('dotenv'); // 读取环境变量

dotenv.config();

// console.log(process.env.APP_PORT);

/**
 * @process 当前node进程
 * @env 环境变量
 */
module.exports = process.env;
    1. 在main.js中引入
    const Koa = require('koa');
const { APP_PORT } = require('./config/config.default')

const App = new Koa();

App.use((ctx, next) => {
  ctx.body = "hello"
})

App.listen(APP_PORT, () => {
  console.log(`serve is running on http://localhost:${APP_PORT}`);
})

    # 添加路由---写接口

    路由：根据不同的URL，调用对应的处理函数

    # 安装koa-router

    npm install koa-router

    koa-router文档 (opens new window)

    1. 编写路由

    创建src/router目录，编写user.route.js，实现用户登录注册等功能

    const Router = require('koa-router')
const router = new Router({prefix: '/users'}); // prefix 当前模块接口统一前缀

// GET /users/
router.get('/', (ctx, next) => {
  ctx.body = '返回用户信息'
})

module.exports = router;
    1. 在main.js中引入
    const userRouter = require('./router/user.route')
App.use(userRouter.routes());

    在浏览器中输入http://localhost:8001/users 页面中返回信息，至此第一个get请求编写完成

    # 贴合业务，进行项目目录结构拆分

    在实际项目中，这么去编写代码是不够贴合实际的，为了让结构更清晰，所以需要将各种功能模块进行拆分

    1. 将http服务和app业务拆分

    创建src/app/index.js 业务部分 main.js只作为服务主入口

const userRouter = require('../router/user.route')
App.use(userRouter.routes());

    改写main.js

})
    1. 路由与控制器拆分

    在路由文件中，可以将业务处理的函数，放入到控制器中，由控制器来管理具体业务逻辑的处理，这样路由文件可以专注于解析URL，分发给控制器对应的方法

    新建controller/user.controller.js,用类的方式创建控制器

    class UserController {
  async register(ctx, next) {
    ctx.body = '用户注册成功'
  }

  async login(ctx, next) {
    ctx.body = '登录成功'
  }
}

module.exports = new UserController();

    改写user.route.js

    const Router = require('koa-router') // 导入中间件
const { register, login } = require('../controller/user.controller')

const router = new Router({prefix: '/user'}); // prefix 当前模块接口统一前缀

// POST /users/register 注册接口
router.post('/register', register)

// 登录接口
router.post('/login', login)

module.exports = router;

    # 数据库操作

    在node中操作数据库，这里我们使用的是Sequelize中间件，Sequelize是一个基于 promise 的 Node.js ORM,目前支持 Postgres, MySQL, MariaDB, SQLite 以及 Microsoft SQL Server. 它具有强大的事务支持, 关联关系, 预读和延迟加载,读取复制等功能。 Sequelize文档 (opens new window)

    ORM:对象关系映射

    • 数据表映射（对应）一个类
    • 数据表中的数据行（记录）对应一个对象
    • 数据表字段对应的属性
    • 数据表的操作对应对象的方法

    安装

    npm i sequelize
npm i mysql2

    建立db/seq.js文件 连接数据库 环境变量在.env文件

    # 参数验证 抛出错误

    在controller层中接受参数 进行参数验证 抛出相应错误

    ctx.status 一般设为200 为保证前端浏览器能正常解析，不至于控制台报红，其中业务错误在返回信息中抛出 只有其他非业务错误可由浏览器抛出 例如网络问题、跨域问题、服务器内部问题等

    async register(ctx, next) {
  const { user_name, password } = ctx.request.body;
  // 验证合法性、合理性
  if (!user_name || !password) {
    ctx.status = 200; // 请求格式出错 
    ctx.body = {
      code: '10001',
      message: '用户名或者密码不能为空',
      result: ''
    }
    return
  }
  if (getUserInfo({user_name})) {
    ctx.status = 200;
    ctx.body = {
      code: '10002',
      message: '用户名已存在',
      result: ''
    }
    return
  }
  const res = await createUser(user_name,password)
  console.log(res,"数据库");
  ctx.body = {
    code: 200,
    message: "用户注册成功",
    result: {
      id: res.id,
      user_name: res.user_name
    }
  };
}

    上述代码中的getUserInfo为封装的公共函数，用于查询数据库，获取某些操作，因为涉及 数据库查询操作，所以将该方法封装于service中

    进入service/user.service.js文件 在UserService类中添加方法 这次用到的是findOne()api 具体请查看sequelize文档

    async getUserInfo({id, user_name, password, is_admin}) {
  const whereOpt = {};
  // 短路运算
  id && Object.assign(whereOpt, { id });
  user_name && Object.assign(whereOpt, { user_name });
  password && Object.assign(whereOpt, { password });
  is_admin && Object.assign(whereOpt, { is_admin });
  const res = await User.findOne({
    attributes: ['id', 'user_name', 'password', 'is_admin'], // 要查询的字段名
    where: whereOpt, // where过滤查询 SELECT * FROM user WHERE user_name=xxx
  })
  return res ? res.dataValues : null
}

    认识koa处理错误信息emit函数和on函数 在app.use()函数内，接收两个参数，ctx可以理解为上下文，next执行下一步操作,若是没有执行next步骤，后面的函数将不会执行 ctx.app.emit()此函数将错误信息传给app.on() app.on()监听并返回对应错误信息

    在app文件夹下添加errHandler.js文件，处理信息的抛出

    module.exports = (err, ctx) => {
  let status = 500;
  switch (err.code) {
    case '10001':
      status = 400;
      break;
    case '10002':
      status = 409;
      break;
    default:
      status = 500;
  }
  ctx.status = status;
  ctx.body = err;
}
